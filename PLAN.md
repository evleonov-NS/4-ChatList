# План реализации ChatList

Поэтапный порядок работ на основании [PROJECT.md](PROJECT.md).

---

## Этап 1. Подготовка проекта

1. Обновить структуру каталогов:
   ```
   4-ChatList/
   ├── main.py
   ├── db.py
   ├── models.py
   ├── network.py
   ├── requirements.txt
   ├── .env.example
   └── docs/
   ```
2. Зафиксировать зависимости в `requirements.txt`: PyQt6, httpx (или requests), python-dotenv.
3. Добавить `.env.example` с шаблоном переменных для API-ключей.
4. Убедиться, что `.env` указан в `.gitignore`.

---

## Этап 2. База данных (`db.py`)

1. Реализовать подключение к SQLite (один файл, например `chatlist.db`).
2. Создать функцию инициализации схемы (`init_db`) по [DATABASE.md](DATABASE.md).
3. Инкапсулировать CRUD-операции для таблиц:
   - `prompts` — добавление, выбор, поиск, сортировка;
   - `models` — добавление, редактирование, получение активных (`is_active = 1`);
   - `results` — сохранение выбранных строк, чтение истории;
   - `settings` — чтение и запись настроек.
4. Добавить контекстный менеджер или единую точку доступа к соединению.
5. Проверить создание БД и базовые операции вручную или простым тестовым скриптом.

---

## Этап 3. Конфигурация моделей (`models.py`)

1. Описать структуру данных модели (dataclass или TypedDict): name, api_url, api_id, env_key, is_active, provider.
2. Реализовать загрузку списка моделей из БД через `db.py`.
3. Реализовать получение API-ключа из `.env` по имени переменной (`env_key`).
4. Добавить валидацию: активная модель должна иметь URL, id и непустой ключ в окружении.
5. Подготовить начальный набор моделей (seed) для первого запуска — OpenAI, DeepSeek, Groq и т.д.

---

## Этап 4. Сетевой слой (`network.py`)

1. Реализовать базовый интерфейс отправки промта в модель.
2. Добавить адаптеры под разные провайдеры (OpenAI-compatible, DeepSeek, Groq).
3. Отправлять промт параллельно во все активные модели (async httpx или потоки).
4. Обрабатывать ошибки сети и API: таймауты, неверный ключ, пустой ответ.
5. Возвращать единый формат результата: `{model_name, response_text, error?}`.

---

## Этап 5. Основной интерфейс — каркас (`main.py`)

1. Разбить UI на логические блоки:
   - панель ввода промта;
   - выбор сохранённого промта из БД;
   - кнопка «Отправить»;
   - таблица временных результатов;
   - кнопка «Сохранить».
2. Создать класс главного окна `MainWindow`.
3. При запуске вызывать `init_db()` и загружать список промтов и моделей.

---

## Этап 6. Рабочий процесс — ввод и отправка промта

1. **Ввод промта:** текстовое поле + выпадающий список сохранённых промтов из `prompts`.
2. При выборе сохранённого промта — подставлять текст в поле ввода.
3. По нажатию «Отправить»:
   - взять активные модели из `models.py`;
   - вызвать `network.py` для каждой модели;
   - очистить предыдущую временную таблицу в памяти;
   - заполнить новую временную таблицу результатами.
4. Показать индикатор загрузки на время запросов.

---

## Этап 7. Таблица временных результатов

1. Отобразить таблицу (QTableWidget или QTableView + модель):
   - колонка «Модель»;
   - колонка «Ответ»;
   - колонка «Выбрать» (чекбокс `selected`).
2. Хранить временные результаты только в памяти (список объектов), не в SQLite.
3. При новом промте — полностью удалять старые временные данные.

---

## Этап 8. Сохранение выбранных результатов

1. По нажатию «Сохранить»:
   - отфильтровать строки с `selected = True`;
   - записать их в таблицу `results` через `db.py`;
   - при необходимости сохранить новый промт в `prompts`.
2. Очистить временную таблицу после успешного сохранения.
3. Показать пользователю подтверждение или сообщение об ошибке.

---

## Этап 9. Управление моделями и настройками (UI)

1. Добавить экран/диалог настройки моделей:
   - список моделей;
   - включение/выключение (`is_active`);
   - редактирование name, api_url, api_id, env_key.
2. Добавить экран настроек программы (таблица `settings`).
3. Реализовать просмотр сохранённых результатов из `results`.

---

## Этап 10. Дополнительные функции (опционально)

1. Поиск и сортировка во всех таблицах UI.
2. Экспорт выбранных результатов в Markdown / JSON.
3. Логирование запросов в файл (`logs/`).
4. Улучшение UX: теги промтов, фильтр по тегам, копирование ответа в буфер.

---

## Этап 11. Сборка и документация

1. Обновить `README.md` — описание программы, установка, настройка `.env`.
2. Проверить сборку `.exe` через PyInstaller ([docs/run-and-build.md](docs/run-and-build.md)).
3. Прогнать сценарий вручную:
   - первый запуск → создание БД;
   - отправка промта в 2+ модели;
   - выбор строк → сохранение;
   - новый промт → временная таблица обнуляется.
